# Task Manager CLI

A lightweight command-line task management tool that helps developers
create, prioritise, and track tasks directly from the terminal.

## Features

- **Task Creation** — Create tasks with titles, priorities, due dates, and tags
- **Smart Prioritisation** — Automatic scoring system ranks tasks by urgency
- **Status Tracking** — Move tasks through TODO, IN_PROGRESS, REVIEW, and DONE
- **Natural Language Input** — Create tasks using shortcuts like `!urgent` and `#tomorrow`
- **CSV Export** — Export your task list to a CSV file for reporting or sharing
- **Overdue Detection** — Automatically identifies and abandons long-overdue tasks
- **Persistent Storage** — Tasks saved to a local JSON file between sessions

## Technologies Used

- Node.js (v14 or higher)
- Commander.js — CLI argument parsing
- File system (fs) — Local JSON storage
- No external database required

## Installation

### Prerequisites
- Node.js 14.0 or higher
- npm (comes with Node.js)

### Steps

1. Clone or download the repository:
```bash
git clone https://github.com/Kwa8t4k/ai-code-exercises.git
cd ai-code-exercises
```

2. Install dependencies:
```bash

```

3. Verify installation:
```bash
node cli.js --help
```

## Usage

### Create a task
```bash
node cli.js add "Write unit tests"
node cli.js add "Fix login bug !urgent #tomorrow @backend"
```

### List all tasks
```bash
node cli.js list
```

### Update a task status
```bash
node cli.js update <taskId> --status IN_PROGRESS
```

### Mark a task as complete
```bash
node cli.js complete <taskId>
```

### Export tasks to CSV
```bash
node cli.js export
```

### Check for overdue tasks
```bash
node cli.js check-overdue
```

## Natural Language Task Syntax

When creating tasks, you can use shortcuts in the task title:

| Shortcut | Meaning | Example |
|----------|---------|---------|
| !1 or !low | Low priority | "Review docs !low" |
| !2 or !medium | Medium priority | "Update readme !medium" |
| !3 or !high | High priority | "Fix crash !high" |
| !4 or !urgent | Urgent priority | "Deploy hotfix !urgent" |
| @tagname | Add a tag | "Fix bug @backend" |
| #tomorrow | Due tomorrow | "Review PR #tomorrow" |
| #friday | Due next Friday | "Submit report #friday" |
| #YYYY-MM-DD | Specific due date | "Launch #2025-01-15" |

## Configuration

No configuration file is required. Tasks are stored automatically in `tasks.json`
in the project root directory.

To reset all tasks, delete the `tasks.json` file:
```bash
del tasks.json    (Windows)
rm tasks.json     (Mac/Linux)
```

## Troubleshooting

### "Cannot find module" error
Run `npm install` to ensure all dependencies are installed.

### Tasks not saving between sessions
Check that `tasks.json` exists in the project root and that you have
write permissions to that directory.

### Export file not appearing
Run `node cli.js export` from the project root directory.
The file `tasks-export.csv` will be created in the same folder.

## Prompt 2 Output: Step-by-Step Guide

### How to Create and Export Tasks Using the Task Manager CLI

**Target audience:** Beginner developers new to command-line tools.

---

#### Prerequisites
Before starting, make sure you have:
- Node.js installed (check with `node --version` in your terminal)
- The project cloned and `npm install` completed
- A terminal open in the project root folder

---

#### Step 1 — Open your terminal in the project folder

On Windows in VS Code, press **Ctrl + `** (backtick) to open the terminal.
Confirm you are in the right folder by running:
````bash
dir
````
You should see `cli.js` in the file list. If not, navigate to the correct folder first.

---

#### Step 2 — Create your first task

Run this command:
````bash
node cli.js add "Buy groceries"
````

You should see a confirmation message with the new task's ID.
The task is automatically saved to `tasks.json`.

---

#### Step 3 — Create a task with priority and due date

You can include extra information directly in the task title:
````bash
node cli.js add "Submit assignment !high #tomorrow"
````

This creates a task with:
- Title: "Submit assignment"
- Priority: HIGH
- Due date: tomorrow's date

**Common mistake:** The `!` and `#` shortcuts must have a space before them.
`"Submit assignment!high"` will NOT be parsed — use `"Submit assignment !high"`.

---

#### Step 4 — View all your tasks
````bash
node cli.js list
````

Tasks are displayed sorted by their priority score, with the most urgent at the top.

---

#### Step 5 — Update a task's status

Copy the task ID from the list output, then run:
````bash
node cli.js update <taskId> --status IN_PROGRESS
````

Available status values: TODO, IN_PROGRESS, REVIEW, DONE

---

#### Step 6 — Mark a task as complete
````bash
node cli.js complete <taskId>
````

The task status changes to DONE and it will move to the bottom of future lists.

---

#### Step 7 — Export all tasks to CSV
````bash
node cli.js export
````

A file called `tasks-export.csv` is created in the project folder.
You can open it in Excel or Google Sheets.

---

#### Troubleshooting This Process

**Problem:** "node is not recognized as a command"
**Solution:** Node.js is not installed or not added to your PATH.
Download it from nodejs.org and reinstall.

**Problem:** Task ID not found when updating
**Solution:** Copy the exact ID from the `list` output. IDs are case-sensitive.

**Problem:** CSV file is empty
**Solution:** Make sure you have created at least one task before exporting.

## Prompt 3 Output: FAQ Document

### Task Manager CLI — Frequently Asked Questions

---

#### Getting Started

**Q: What do I need installed before using this tool?**
You need Node.js version 14 or higher. Download it free from nodejs.org.
Once installed, run `npm install` in the project folder before your first use.

**Q: How do I know if the tool is set up correctly?**
Run `node cli.js --help` in your terminal. If you see a list of available commands,
the setup is working correctly.

**Q: Where are my tasks stored?**
Tasks are saved automatically in a file called `tasks.json` in the project root folder.
This file is created the first time you add a task. You can open it in any text editor.

---

#### Creating and Managing Tasks

**Q: How do I create a task with a priority?**
Include a priority shortcut at the end of your task title:
````bash
node cli.js add "Fix the bug !urgent"
node cli.js add "Update docs !low"
````
Options are: !low, !medium, !high, !urgent (or !1 through !4)

**Q: Can I set a due date when creating a task?**
Yes. Use a # followed by a date keyword or specific date:
````bash
node cli.js add "Submit report #friday"
node cli.js add "Pay invoice #2025-02-28"
````
Supported keywords: today, tomorrow, next_week, and any weekday name (monday, tuesday, etc.)

**Q: How do I add tags to a task?**
Include @tagname in the task title:
````bash
node cli.js add "Fix login @backend @urgent"
````

**Q: Can I edit a task after creating it?**
You can update the status using the update command with the task ID.
To change the title or other fields, you would need to delete and recreate the task.

---

#### Understanding Task Priority

**Q: Why is a LOW priority task appearing at the top of my list?**
The list is sorted by a calculated score, not just the priority label.
A LOW priority task that is overdue scores higher than a HIGH priority task
with no due date. This is intentional — overdue tasks need attention.

**Q: What does the score number next to each task mean?**
The score combines priority, due date urgency, tags, and recent activity
into one number. Higher score = appears higher in the list.
You do not need to manage this number — it is calculated automatically.

**Q: What happens to tasks marked as DONE?**
Completed tasks receive a large score penalty and sink to the bottom of the list.
They remain in your task data and appear in CSV exports.

---

#### Exporting and Data

**Q: How do I export my tasks?**
Run `node cli.js export` from the project root. This creates `tasks-export.csv`
which can be opened in Excel or Google Sheets.

**Q: How do I start fresh with no tasks?**
Delete the `tasks.json` file from the project folder. A new empty one will be
created automatically when you add your next task.

**Q: Can I use this tool on multiple computers?**
Not automatically — tasks are stored locally. To share tasks between computers,
copy the `tasks.json` file manually or store the project in a shared folder.

---

#### Troubleshooting

**Q: I get an error saying "Cannot find module"**
Run `npm install` in the project folder. This installs missing dependencies.

**Q: My task was not saved after I created it**
Make sure you ran the command from inside the project folder where `cli.js` lives.
Check whether `tasks.json` was created or updated after your command.

**Q: The CSV export is empty or missing tasks**
Confirm you have tasks by running `node cli.js list` first.
Then run the export command again from the project root directory.

## Reflection

### Most challenging part to document
Writing the README installation section required thinking carefully about
what a complete beginner would not know — things like "what is a terminal"
or "what does npm install do". Specifying "beginner audience" in the prompt
helped the AI include those explanations.

### How I adjusted my prompts
For the step-by-step guide, I added "include common mistakes after each step"
to the prompt. This produced much more practical guidance than the default output,
which only listed happy-path steps.

### What I learned about document structure
Good documentation answers three questions in order:
what is this, how do I set it up, and how do I use it.
The AI naturally follows this structure when given enough project context.

### How I would use this in my workflow
I would run the README prompt at the start of every new project,
even before the code is finished, to force myself to think about
what the project does and how someone else would use it.
